## **代收API**

### **代收申请**

**简要描述：**

* 创建代收订单

**此地址永久有效，对接时请先查看 [注意事项](#注意事项)**

**URL：**

USDT

`/usdt/collect/apply/multiple`
说明：appId USDT，实收结算USDT

**参数：**

| 参数名        | 必选 | 类型     | 说明                              |
|:-----------|:---|:-------|:--------------------------------|
| appId      | 是  | Long   | 业务ID（后台获取，需要根据URL中的币种传递对应的业务ID） |
| payType    | 是  | String | [支付方式类型](#inBankCode)           |
| inBankCode | 是  | String | [支付方式编码](#inBankCode)           |
| sign       | 是  | String | 签名                              |

返回data参数说明

| 参数名    | 类型     | 说明   |
|:-------|:-------|:-----|
| payUrl | String | 支付链接 |
| bankNo | String | 链地址  |
| sign   | String | 签名   |

### **异步通知**

**URL：`贵方提供`**

参数：

| 参数名      | 类型      | 说明                                    |
|:---------|:--------|:--------------------------------------|
| appId    | Long    | 业务ID                                  |
| currency | String  | 币种                                    |
| orderId  | String  | 商户订单号，每次回调单号不同                        |
| orderNo  | String  | 平台订单号，每次回调单号不同                        |
| amount   | String  | 本次收款金额                                |
| fee      | String  | 手续费金额                                 |
| status   | Integer | 状态(2收款成功)                             |
| payTime  | String  | 支付成功时间(纽约时间), 格式: yyyy-MM-dd HH:mm:ss |
| hash     | String  | 交易hash                                |
| sign     | String  | 签名                                    |

request

```json
{
  "appId": 10002,
  "currency": "USD",
  "orderId": "M123456789",
  "orderNo": "PLAT123456789",
  "amount": "21.00",
  "fee": "1.00",
  "status": 2,
  "payTime": "2024-03-01 23:59:59",
  "sign": "SIGN123456789"
}
```

备注：收到回调通知之后,请返回**SUCCESS**(**大写**) 否则会持续通知5次

### **注意事项**

1，下单后的地址永久有效，每次支付后，都会触发回调，回调时，平台订单号(orderNo)和商户订单号(orderId)每次均不一样。但**同一笔订单**重复回调（5次重试通知）是相同的

2，如果已存在某网络协议对应的收款地址，则会将请求中的**currencies**更新到已绑定的地址中更新方式为**覆盖**，原币种将不再支持，请谨慎操作

3，仅支持下单时选定的 [**网络协议与币种**](#inBankCode)，未选择的不支持支付，强行支付资金无法入账，也不会有回调通知，为避免资金损失，请确认后支付
